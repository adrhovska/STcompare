# STworkflow README

STworkflow.sh is a bash script which runs the full workflow pipeline for comparing two 10x Visium ST tissue sections: LandmarkPicker, STalign and STcompare, run in that order. It aligns a source tissue to a reference tissue using manually picked H&E landmarks (LandmarkPicker.py and STalignCode.py), then runs the downstream spatial comparison (STcompare.R). It uses two primary languages, python for LandmarkPicker and STalignCode and R for STcompare. The STalign parameters need to be tuned per sample pair to get a good fit, as there is no universal that works everywhere (see below how to troubleshoot). Such can be optimised by inspecting the alignment QC plots which are generated by a separate python script STalignQC.py.

Each sample needs to be a standard 10x Visium output folder in order for the script to work smoothly:

```text
Sample_ST/
|--- filtered_feature_bc_matrix.h5
|___ spatial/
    |--- tissue_hires_image.png
    |--- tissue_positions.csv
    |___ scalefactors_json.json
```

Two separate conda environments are used: `python_env` for LandmarkPicker.py and STalignCode.py, `r_env` for STcompare.R. Override either with `--py_env` / `--r_env` if your local env names differ.

Make the script executable once (`chmod +x STworkflow.sh`), then run it:

```bash 
./STworkflow.sh \
  --source_dir /path/to/source_folder \
  --reference_dir /path/to/reference_folder \
  --sample_aligned SourceSample \
  --sample_reference ReferenceSample
```

**Four required arguments:** `--source_dir` (source 10x Visium folder, the one being aligned), `--reference_dir` (reference 10x Visium folder, the alignment template), `--sample_aligned` (name of the aligned sample), `--sample_reference` (name of the reference sample)

**Optional arguments:** `--project_dir` (output project directory, defaults to cwd), `--script_dir` (directory containing LandmarkPicker.py, STalignCode.py, STcompare.R, defaults to the directory STworkflow.sh is in), `--py_env`, `--r_env`, `--counts1` (custom source counts h5, defaults to `SOURCE_DIR/filtered_feature_bc_matrix.h5`), `--counts2` (custom reference counts h5, defaults to `REFERENCE_DIR/filtered_feature_bc_matrix.h5`), `--spatial2` (custom reference spatial folder, defaults to `REFERENCE_DIR/spatial`)

# Outputs 
The workflow creates one pair-specific output folder containing Landmarks, STalign and STcompare subfolders. Landmarks holds `rsource_points.csv` and `reference_points.csv`, the two essential files generated by LandmarkPicker.py and passed into STalignCode.py as `--points1` / `--points2`, plus a combined pair CSV used only for QC (to compare whether the order of the landmarks and their overall number matches between columns). STalign includes the aligned barcode CSV and the affine/LDDMM transform file, plus a full set of QC plots when --skip_stalign_qc isn't passed: before/after alignment overlays, the landmark fit plot, input H&E with landmarks, initial affine, deformed source vs target, deformation grid, aligned spots on target, spot displacement histogram, LDDMM diagnostic panel, WM values on spots and their histogram, and a combined QC summary panel combining all of the above into one image. STcompare holds Results, Coordinate_QC, Raster_Plots, Correlation_Plots, Linear_Regression, and Pixel_Class. The main result table is `STcompare/Results/Results_Table.csv`, and `Results/` also contains `Summary_Report.pdf`, a one-page overview PDF (more on what's in it further down).

**Note:** landmarks are picked on`tissue_hires_image.png`, so STalignCode.py converts full-resolution Visium spot coordinates into hires coordinates to keep landmarks and spots in the same space. The workflow always runs LandmarkPicker.py fresh, so every run generates a new set of landmarks and reusing the same sample names and output folder will overwrite previous landmark and alignment outputs. Nothing project-specific is hardcoded; all paths come through CLI args and outputs land in cwd or whatever `--project_dir` you pass.

# LandmarkPicker.py
LandmarkPicker.py is the manual landmark-selection step. It opens the source and reference H&E images sequentially, source first, then reference, and has you click matching landmark points between them, one pair at a time, for however many pairs you asked for at the start (6-10 recommended). Pick landmarks on morphological grounds such as equivalent anatomical features, or fallback signatures like indentations, tissue corners, folds, internal cavities/lumen, and spread them across the whole tissue rather than clustering them in one spot/on one side.

Needs Python 3.9+ and `pandas, matplotlib (pyplot and image), argparse, pathlib`

Run it as:

```text
python LandmarkPicker.py \
  --image1 /path/to/source_sample/spatial/tissue_hires_image.png \
  --image2 /path/to/reference_sample/spatial/tissue_hires_image.png \
  --sample_aligned SourceSample \
  --sample_reference ReferenceSample
```

**Four required arguments:** `--image1`/`--image2` (source/reference hires PNGs), `--sample_aligned`, `--sample_reference`

**Optional arguments:** `--project_dir` to override where the landmark output folder gets created. Outputs land in a directory named after both samples, containing the source landmark CSV, reference landmark CSV, and a combined CSV for QC with only the first two actually needed downstream, the combined one just lets you check landmark order was preserved (landmark 1 in tissue A actually corresponds to landmark 1 in tissue B). Note: matplotlib records clicks as x,y, but this script deliberately flips and saves them as y,x, because that's the order STalignCode.py expects for `--pos1`/`--pos2`

# STalignCode.py
STalignCode.py takes those landmark pairs and aligns the source tissue to the reference. Firstly, it fits an affine transform from the landmarks, then also runs STalign's nonlinear LDDMM registration on top for a proper deformable fit (recommended over affine-only, which can't handle local warping - only translation, rotation, scaling, shearing). The fitted transform gets applied to all source Visium spot coordinates, producing an aligned coordinate file in the reference's coordinate space, ready for analysis using STcompare.

Needs Python 3.9 and `numpy, pandas, matplotlib, argparse, json`

Run it as:

```text
python STalignCode.py \ 
    --pos1 /path/to/source_sample/spatial/tissue_positions.csv \ 
    --pos2 /path/to/reference_sample/spatial/tissue_positions.csv \ 
    --scale1 /path/to/source_sample/spatial/scalefactors_json.json \ 
    --scale2 /path/to/reference_sample/spatial/scalefactors_json.json \ 
    --points1 /path/to/landmarks/source_points.csv \ --> from LandmarkPicker
    --points2 /path/to/landmarks/reference_points.csv \ --> from LandmarkPicker
    --sample_aligned SourceSample \ 
    --sample_reference ReferenceSample 
```

Landmark files must have matching pairs in the same row order — row 1 in points1 corresponds to row 1 in points2, and so on (LandmarkPicker.py already guarantees this). Expected CSV format:

```text
y,x
320.5,510.2
480.1,1300.7
900.4,1500.3
1450.2,1200.8
1600.6,650.1
850.9,300.
```

**Eight required args:** `pos1`, `pos2` (source/reference tissue_positions.csv), `scale1`, `scale2` (source/reference scalefactors json), `points1`, `points2` (source/reference landmark CSVs), `sample_aligned`, `sample_reference`

**Optional:** `project_dir`, `outdir` (custom output dir), `outname` (custom name for the aligned barcode CSV).

The LDDMM parameters need per-sample tuning or the nonlinear fit can go wrong or fail outright (empty output, crashing diagnostic plots, infinite values):

-`sigmaM` (default 1.0) is the noise SD for the image-matching term — smaller pulls harder toward matching intensities
-`sigmaB` (default 2.0) is the same idea for background, kept larger since matching background matters less than matching tissue
-`sigmaA` (default 5.0) is for artefacts, kept largest since artefacts should have the weakest pull
-`sigmaP` (default 20.0) controls how hard the manually clicked landmarks are enforced — too small (under ~1) makes them pull too aggressively and destabilizes the optimization
-`epV` (default 1.0) is the step size for velocity field updates — if you see NaNs or blown-up deformation, this is the first one you should lower
-`niter` (default 1000) is total iterations
-`diffeo_start` (default 100) is the iteration nonlinear deformation kicks in after an affine-only warm-up

**Important**: the sigmas act roughly as `1/sigma²` in the loss function, so tightening `sigmaP` leads to small changes having an outsized nonlinear effect, so adjust in small steps.

**If something breaks:** reset sigmas to the library defaults first (`sigmaM=1, sigmaB=2, sigmaA=5, sigmaP=20`), then try lowering `epV` toward 1, then check landmark placement, because if too clustered, too few (fewer than ~6), or clicked in mismatched order between source and reference can produce a bad initial affine that poisons the LDDMM fit. Landmarks are picked interactively each run, so results can vary slightly even with identical parameters — check the QC4 landmark fit plot before reaching for parameter changes. For QC, the script prints the fitted affine matrix, landmark residuals (mean/median/max), affine scale singular values, and the affine determinant, check that residuals are small, the determinant is positive (negative means the image got flipped), and the singular values aren't extreme in either direction.

Ultimately, you should troubleshoot in the order it runs: landmarks --> affine --> LDDMM.

# STcompare.R
STcompare.R is the downstream comparison step, which takes the two 10x Visium samples (one now aligned into the other's coordinate space) and compares spatial pattern and expression magnitude for genes specific to three tissue types which you have changed in the code accrdingly to your research question: epithelium, smooth muscle, skeletal muscle. This allows e.g. checking how similar an engineered tissue sample is to native tissue.

Needs R 4.2+ and the packages Seurat, ggplot2, patchwork, jsonlite, scales, gridExtra, plus SpatialExperiment, SEraster, STcompare and SummarizedExperiment from BiocManager

Run it as:

```text
Rscript STcompare.R \
  --counts1  /path/to/aligned_sample/filtered_feature_bc_matrix.h5 \
  --counts2  /path/to/reference_sample/filtered_feature_bc_matrix.h5 \
  --pos1     /path/to/aligned_sample(has to include barcodes).csv \
  --spatial2 /path/to/reference_sample/spatial \
  --outdir   /path/to/output_folder \
  --sample_aligned   (insert name) \
  --sample_reference (insert name)
```

**Four required args:** `--counts1`, `--counts2` (Cell Ranger .h5 count matrices for aligned/reference samples), `--pos1` (CSV with columns barcode, x, y for the aligned sample), `--spatial2` (reference sample's spatial/ folder)

**Optional:** `--outdir`, `--sample_aligned`, `--sample_reference`, `--scale` (hires or lowres image resolution), `--res` (raster grid resolution — higher is coarser, lower is finer; default is 50, can be imagined as the size of the rasters), `--threads` (CPU cores). Outputs land in a directory named after both samples with subfolders for: results (correlation coefficients + p-values per gene), coordinate QC (overlap check for scaling/alignment correctness), raster plots, correlation scatter plots (matched pixel expression values), linear regression plots, and a pixel-class spatial map colored by which sample predominantly expresses a gene above threshold (threshold is the mean gene expression in that sample).

Once every gene's been rastered, correlated and plotted, the script also builds one summary PDF, `Results/Summary_Report.pdf`. It contains a text panel (sample names, matched spot counts, number of genes analysed, number significant at p < 0.05, mean/median correlation, best- and worst-correlated genes with their r values), a small table of `correlationCoef` and `empirical_pval` per gene rounded to 3 decimals, and the raster pair + correlation scatter plot reused directly from the best-correlated gene's own output. It also throws in a rough overall-fit call based on the mean correlation across all tested genes: mean r ≥ 0.6 is called strong spatial correspondence, ≥ 0.3 is moderate ("some markers align well, others may need review"), below that is weak ("try reviewing alignment quality first, check QC outputs from STalign"). This is a coarse heuristic over the whole gene panel, not a verdict on any individual gene, for that check `Results_Table.csv` and the per-gene plots directly before drawing conclusions.

## Troubleshooting
If you're getting a missing file error, check the source/reference folders actually contain `filtered_feature_bc_matrix.h5`, `spatial/tissue_hires_image.png`, `spatial/tissue_positions.csv`, `spatial/scalefactors_json.json`
If landmark files aren't found, check LandmarkPicker.py saved the landmark folder using the expected naming pattern `SampleAligned_paired_to_SampleReference` and that the workflow is pointing at that same folder name.
If conda activation fails, make sure conda's actually available in the terminal and that the env names passed via `--py_env`/`--r_env` match your real local env names. 
If you get no matching barcodes, check the count matrix and position files actually come from the same sample — barcode suffix differences like `-1` are handled automatically in STcompare.R, but genuinely mismatched files will still fail.
If few genes are found in both samples, check whether the genes in `genes_of_interest` (inside STcompare.R) are actually present in both count matrices.

## Current limitations
This workflow only does global affine + LDDMM alignment from manually picked landmarks and quality depends heavily on how good and well-distributed those landmarks are, so keep them clearly identifiable in both samples and spread across the full tissue section rather than clustered. The gene panel in STcompare is hardcoded to `genes_of_interest` inside STcompare.R, based on https://doi.org/10.1038/s41587-026-03043-1, edit that list directly if you want different genes.

**On reproducibility:** each run saves the picked landmarks, the fitted affine transform, the aligned coordinate CSV, and all STcompare outputs together in one pair-specific output folder, so any comparison can be traced back to the exact landmarks and transform that produced it. Reusing the same sample names and output directory will overwrite the previous run's outputs, so use a new `--project_dir` or rename the output folder first if you want to keep it.
